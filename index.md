### Agent / 软件工程
- `[[concepts/Code-as-Liability]]` — 代码即负债：代码一旦进入生产环境首先应被视为负债，能否转化为资产取决于是否对业务客户产生正向价值；阿里云 CIO 蒋林泉提出
- `[[concepts/Half-Stack]]` — Half-Stack 岗位设计：不追求稀缺全栈工程师，利用 AI 降低跨域门槛，将职能合并为 PDFE（产品/设计/前端）与 ABE（架构/后端/Agent）两类端到端角色
- `[[concepts/Intent-Alignment]]` — 意图对齐：在有限 context 与不均匀注意力下，将意图从对话中外化为结构化文档的工程问题
- `[[concepts/Product-Tri-Ownership]]` — 产品端到端三人组框架：PO（做什么）+ QO（做得对不对）+ TO（怎么做）+ AI Agent（施工方），让普通团队实现 AI 原生开发
- `[[concepts/Agent-Self-Drive]]` — Agent 自驱闭环：先定义成功（验收标准+自验证）+ 结果导向（给目标不给步骤），让 Agent 从"踢一脚动一下"变自驱
- `[[concepts/Goal-Context-Constraints]]` — Agent 时代思考三元组（目标-上下文-约束）
- `[[concepts/Agent-Infra]]` — Agent 友好基础设施的三大设计原则
- `[[concepts/Agent-Control-Plane]]` — Agent 企业级治理基础设施：注册、身份、审计、计费、权限
- `[[concepts/Harness-Engineering]]` — 上下文工程的具体实践领域：三支柱模型（上下文工程/架构约束/持续治理），已加入 cache discipline、模型接入代价、知识工程维度、OpenAI 四维度实践、Spec-Anchored-Verification 验证维度
- `[[concepts/SDD]]` — Spec-Driven Development，规格驱动开发：用结构化文档链约束 AI 产出，需求规格→架构设计→接口定义→开发计划的逐层收敛
- `[[concepts/Spec-Anchored-Verification]]` — 以规约为锚点的验证：spec 是验证的 single source of truth，测试验证行为正确性，code review 验证语义完整性，两者互补缺一不可
- `[[concepts/Vibe-Engineering]]` — AI 原生研发的更高维度范式：组织形态、人机分工、认知迭代与工程系统进化；1个月认知半衰期、5万行阈值、90%验收瓶颈、头狼+狼群组织
- `[[concepts/Vibe-Coding]]` — AI 自主写代码的编程范式（Claude Code/Codex CLI/Cursor），本质是开环控制；头狼模式是其精英个体实践，PTO 框架是其团队规模化路径
- `[[concepts/代理工程]]` — 代理工程（Agentic Engineering）：协调强大AI代理以达更高生产力，同时维持专业软件质量标准的新兴工程学科；与"氛围编码"形成对照；卡帕西提出
- `[[concepts/Result-Certainty]]` — 结果确定性：从"规定每一步怎么做"转向"规定终点长什么样、怎么验证"，AI 时代的新安全感来源
- `[[concepts/Poor-Self-Evaluation-Bias]]` — 同一 LLM 写代码又检查代码时自欺欺人的偏差，解决方案为强制角色隔离
- `[[concepts/Prompt-Caching]]` — Prompt Caching 作为 Harness 一等架构约束:viability、prefix stability、sub-agent 边界、度量方法论
- `[[concepts/Model-Integration-Cost]]` — 新模型接入成熟 agentic 系统的隐性工程代价(模型行为与系统假设不匹配)
- `[[concepts/Anti-Distillation]]` — 防止 API 输出被竞争对手蒸馏训练的三层技术防线(假工具/摘要遮蔽/协议隔离)
- `[[concepts/MCP]]` — Model Context Protocol，Anthropic 2024 年底提出，已成为跨厂 agent 工具调用事实标准
- `[[concepts/Multi-Model-Orchestration]]` — 跨模型任务派发与组合工作流：单一模型不再覆盖所有任务
- `[[concepts/Client-Stickiness]]` — 客户端粘性：Anthropic 押注客户端入口而非模型层护城河的战略判断
- `[[concepts/Model-Commoditization]]` — 模型商品化：模型层差异化窗口缩短，能力随团队流动而非沉淀于公司

### 落地架构 / 交互设计
- `[[concepts/AI-as-Translator]]` — AI 仅承担自然语言到 API 的翻译角色，数据/逻辑/权限归传统系统
- `[[concepts/Zero-Barrier-Interface]]` — 将 AI 使用门槛降到用户现有习惯水平，从「人适应系统」变成「系统适应人」
- `[[concepts/Skill-Boundary]]` — 通过预定义 Skill/API 集合严格限制 AI 行为范围，消除发散性

### 人机协作 / 技能形成
- `[[concepts/Skill-Formation]]` — 工作中习得新技能的过程：AI 加速 ≠ 能力建设，概念/阅读/调试三类技能在 AI 介入下被实证削弱
- `[[concepts/Cognitive-Offloading]]` — 认知外包:将原本由大脑承担的推理工作让渡给 AI，与"完成任务但没学到东西"的失真感强相关
- `[[concepts/AI-Interaction-Patterns]]` — 六种 AI 交互模式典型学：三种低分(Delegation/Progressive/Iterative-Debug)、三种高分(Gen-then-Compre/Hybrid/Conceptual-Inquiry)

### AI 落地与策略
- `[[concepts/AI合规]]` — AI大模型合规全景图：十大领域（伦理/法律责任/安全技术/行业应用/跨境/隐私/知识产权/内容/算法/训练数据）、三大监管框架（欧盟/美国/中国）、核心原则（合规前置/最小必要/透明适度/持续监控/多方协同）
- `[[concepts/古德哈特定律]]` — Goodhart's Law：当一个指标成为目标，它就不再是一个好指标；AI 时代新指标（Token/Skill 数量）特别容易陷入此陷阱
- `[[concepts/Problem-First-AI-Adoption]]` — 问题优先的 AI 落地策略：从业务痛点反推 AI 角色，而非拿着 AI 找场景；含"集邮癖"反模式
- `[[concepts/North-Star-Metric]]` — 北极星指标：驱动 AI 项目决策的具体、可衡量业务结果
- `[[concepts/Knowledge-Half-Life]]` — 知识半衰期：领域知识过时速率，AI 领域估计 < 1 年，直接影响学习投入回报计算
- `[[concepts/定力]]` — AI 时代的人生哲学：家国定力（技术向善的价值坐标）、内心定力（算法时代的独立思考）、沉潜定力（长年深耕的耐心）；与 Result-Certainty/Cognitive-Offloading 形成互补
